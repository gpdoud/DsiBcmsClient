Boot Camp Management System (BCMS)

BCMS is the system that manages the boot camps for MAX Technical Training and this page describes the various functions students of those boot camps need to know

Checking in and out

To check in or out of BCMS, click the CheckIn/Out menu item. A dialog box will display with a message that the student is currently "checked-in" or "checked-out". There is a Note text box where students can enter messages when coming to class late or leaving class early. No note is needed if checking in on-time or checking-out after class ends. Click the blue button which displays the student will "Check-in" or "Check-out".

A student can click the "CheckIn/Out" menu item to see if they are currently checked in or out.

BCMS tracks the attendance of students. It does so by requiring the students to "check-in" at the beginning of class and "check-out at the end. BCMS will keep track of the times a student is checked in and out.

In the morning boot camps, students are required to check-in by 9:00 AM. BCMS will allow a 15 minute grace period were the students checking-in by 9:15 AM will still be considered on time. After that grace period, a student is considered "late" for class. There are state board requirements for attendance and an excessive number of late arrivals may result in dismissal from the cohort.

Logging in

To log into BCMS, the student must enter their username and password. By default, a student's username is the first initial of their first name plus their entire last name. For example, for a student named Noah Phence, the username would be nphence. The password for all students defaults to Train@MAX.

Students should can change their profile information by clicking their lastname on the right side of the menu bar. To change the password, which is not displayed, the student must enter it twice. The student can change the first and/or last name if needed. If this student's email address is not correct, it should be changed to the correct one. The Cell/Work Phone fields should include the student's cellphone number in the first text box in the form of xxx-xxx-xxxx. Changing the username it NOT recommended.

Taking Assessments

Assessments test that checks a student's knowledge of a general topic presented in the cohort. Passing all assessments is a requirement for graduation.

Most assessments are 10 multiple choice or true and false questions and may include a bonus question. Each question has a point value and there are at least 100 points available. 80 points are required to pass the assessment. If a student does not attain the 80 points, there is a remedial assessment that will be required. Any questions not answered are incorrect.

If the instructor assigns an assessment and the student does not see it in the "Evals" menu item, the student should simply navigate to the "BCMS" menu item then navigate back to "Evals". The assessment should be in the list.

Assessments will be assigned to students and are available under the Evals menu item. The Evaluations List will show all the assessments already taken plus any recently assigned. It shows the number of questions, the time limit, and, if the assessment is complete, the score. Under the Actions column, there will be a link that says Take to take the assessment and Review to review an assessment already completed.

When reviewing an assessment, all the questions and answers are displayed. If a question was answered correctly, the question and the correct answer are displayed in green. If a question was answered incorrectly, the question and the student's answer is displayed in red with the correct answer displayed in green. The point value of the question is displayed on the right.

If a student believes that they answered a question correctly but the question was marked incorrect, the student should notify the instructor.
